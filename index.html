01

Descrição e Modelagem

O Cenário

O problema clássico consiste em 3 hastes (A, B, C) e N discos de tamanhos distintos. O objetivo é mover todos os discos da haste origem (A) para a haste destino (C), usando a haste auxiliar (B).

Restrições Formais

  • Mover apenas um disco por vez.
  • Um disco nunca pode ser colocado sobre um disco menor que ele.
  • Um disco só pode ser movido se estiver no topo de sua haste.
02

Lógica Proposicional

01. Proposições Atômicas

Definimos o estado do sistema através de proposições que indicam a posição de cada disco.

D(i,h) = "Disco i está na haste h"

Exemplos:
D(1,A) = Verdadeiro se Disco 1 está em A
D(2,C) = Falso se Disco 2 não está em C

02. Fórmulas Bem Formadas (FBFs)

Regra de Exclusividade: Um disco não pode estar em dois lugares ao mesmo tempo.

D(1,A) → (¬D(1,B) ∧ ¬D(1,C))
03

Dedução Natural

Validando Movimentos

Utilizamos regras de inferência como o Modus Ponens para provar que um movimento é válido dado o estado atual.

P1

Se o topo de B está livre ou tem disco > D1, posso mover D1.

P2

Fato: O topo de B está livre.

1. (Livre(B) ∨ Maior(Topo(B), D1)) → PodeMover(D1, B)  [Premissa]
2. Livre(B)                                          [Premissa]
3. Livre(B) ∨ Maior(Topo(B), D1)                     [Intro. Disj: 2]
---------------------------------------------------------------------
4. PodeMover(D1, B)                                  [Modus Ponens: 1,3] ✅
04

Lógica de Predicados

∀

Universal

Regras que se aplicam a todos os discos e hastes.

∃

Existencial

Verificar se existe um movimento válido possível.

⊢

Inferência

Derivar novos estados a partir dos axiomas.

A Regra Universal da Torre

"Para todo x, y e h, se x está sobre y na haste h, x não pode ser maior que y."

∀x∀y∀h (Sobre(x,y) ∧ NaHaste(x,h) → ¬Maior(x,y))
05

Substituição e Unificação

Para aplicar as regras gerais (Predicados) a casos específicos do jogo, utilizamos os processos de Substituição e Unificação.

Exemplo de Unificação

Queremos verificar se a regra de movimento se aplica ao Disco 1.

Regra: PodeMover(x, y)
Fato: PodeMover(D1, B)
MGU (σ): { x / D1, y / B }

Aplicação na Regra Universal

Substituindo variáveis por constantes do estado atual:

Fórmula: ∀x∀y (Sobre(x,y) → ¬Maior(x,y))
Substituição: θ = { x / Disco2, y / Disco1 }

Resultado:
Sobre(Disco2, Disco1) → ¬Maior(Disco2, Disco1)
// Verifica se Disco 2 pode estar sobre Disco 1

6.2 Implementação em Python

O código abaixo implementa a lógica do autômato acima. A classe HanoiAutomaton gerencia os estados e valida as transições com base nas regras da Lógica de Predicados.

Funcionalidades Chave

  • Verificação de Estado (Topo da Haste)
  • Validação de Tamanho (Regra Universal)
  • Transição de Estados (Sucesso/Erro)
automato_hanoi.py 
class HanoiAutomaton:
    def __init__(self):
        self.state = 'IDLE'
        self.towers = {'A': [3, 2, 1], 'B': [], 'C': []}

    def process_move(self, disk, source, target):
        print(f"[State: {self.state}] Tentando mover D{disk} de {source} para {target}...")
        
        # Transição 1: Validação
        self.state = 'VALIDATING'
        
        if not self._is_top_disk(disk, source):
            self.state = 'ERROR'
            return "ERRO: Disco não está no topo!"
            
        if not self._is_valid_placement(disk, target):
            self.state = 'ERROR'
            return "ERRO: Disco maior sobre menor!"

        # Transição 2: Execução
        self.state = 'EXECUTING'
        self._move_disk(source, target)
        
        self.state = 'IDLE'
        return "SUCESSO: Movimento realizado."

    def _is_valid_placement(self, disk, target):
        if not self.towers[target]: return True
        return self.towers[target][-1] > disk

# --- Simulação ---
automato = HanoiAutomaton()
print(automato.process_move(1, 'A', 'C')) 
# Output: SUCESSO: Movimento realizado.

print(automato.process_move(2, 'A', 'C'))
# Output: ERRO: Disco maior sobre menor!
